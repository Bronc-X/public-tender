# 用户操作手册

本文面向业务使用者，说明如何在系统里维护资料、上传招标文件、制作商务标和技术标。

## 1. 登录与基础页面

本项目默认面向本地/内网使用。启动前后端后访问：

```text
http://127.0.0.1:5173/
```

左侧主导航包含：

- 工作台/仪表盘
- 文件中心
- 企业资料库
- 商务标项目
- 技术标项目
- 审计/异常中心
- 系统设置

## 2. 首次使用准备

### 2.1 配置系统能力

进入“系统设置”，至少完成：

1. 配置大模型 Key、Endpoint、模型名称。
2. 配置 OCR 或文档解析服务。
3. 在“行业骨架”中确认技术标目录骨架是否符合当前行业。
4. 保存配置并点击连通测试。

### 2.2 维护企业资料库

在资料库中录入或导入：

- 人员资料
- 企业资质
- 项目业绩
- 荣誉奖项
- 财务报告
- 其他证明材料

这些资料会被商务标“资源方案组合”和技术标“事实映射”使用。

## 3. 文件中心

文件中心用于上传、解析、审计和归档资料。

基本流程：

1. 点击“上传文件”。
2. 选择 PDF、DOCX、图片或表格文件。
3. 等待 OCR/文档解析任务完成。
4. 检查解析文本和审计结果。
5. 将有效资料归档到对应资料库。

如果 OCR 状态异常，先去“系统设置”检查 OCR 服务地址和连通状态。

## 4. 商务标制作流程

商务标项目面向投标函、资格审查、商务响应、资料证明和最终 Word 输出。

### 4.1 创建项目

1. 打开“商务标项目”。
2. 点击“新建项目”。
3. 填写项目名称、招标编号、业主等基础信息。
4. 上传招标文件。

### 4.2 提取招标详情

系统读取招标文件并提取项目基础信息。用户需要检查：

- 项目名称
- 招标编号
- 招标人/代理机构
- 工期、地点、资格要求
- 文件格式要求

确认无误后进入下一步。

### 4.3 招标规则解析

系统解析资格条件、评分项、响应要求和废标风险。用户需要重点检查：

- 资格硬门槛是否完整
- 评分项是否遗漏
- 证书、人员、业绩、财务要求是否分类正确
- 是否存在必须响应但未识别的条款

确认后进入“各项指标适配”。

### 4.4 各项指标适配

系统将招标要求与企业资料库进行匹配。用户需要：

1. 查看每条要求的适配结论。
2. 对不准确的匹配手动调整。
3. 对缺失资料标记补充或忽略。

### 4.5 资源方案组合

在这一步确认每条招标要求使用哪份企业资料。常见动作：

- 给人员要求选择具体人员。
- 给业绩要求选择对应项目业绩。
- 给资质要求选择证书。
- 给财务要求选择财报。

确认关联后进入“方案处理确认”。

### 4.6 方案处理确认

这是商务标 Step6 导出前的关键步骤。

操作要求：

1. 逐章查看系统识别出的标书章节。
2. 点击“装配当前章节核心资源”。
3. 勾选需要写入该章节的资料。
4. 如没有可选资料，也可以填写人工备注或编制要求。
5. 点击“确认装配”。
6. 所有章节都出现确认状态后，点击“确认并执行下一步”。

如果没有完成这一步，后面的“标书章节生成”会提示需要先生成并确认数据块。

### 4.7 标书章节生成

进入“标书章节生成”后：

1. 上传投标文件模板，或使用默认模板。
2. 等待系统生成数据块并装配文档。
3. 点击“下载 Word 文档”。

导出成功后，项目会进入“成果定稿输出”。

### 4.8 成果定稿输出

下载最终 Word 文档后，人工检查：

- 章节顺序是否正确
- 企业资料是否写入正确位置
- 表格、证明材料、签章页是否需要人工补齐
- 格式是否符合招标文件要求

## 5. 技术标制作流程

技术标项目面向施工组织、技术方案、响应措施、风险合规和最终技术标文档。

### 5.1 创建技术标项目

1. 打开“技术标项目”。
2. 新建项目。
3. 上传招标文件或绑定共享招标文件。
4. 启动解析。

### 5.2 项目画像

系统提取技术标关键信息。用户检查：

- 项目类型
- 施工地点
- 工期要求
- 质量、安全、环保要求
- 技术规范和响应等级

确认后进入路线规划。

### 5.3 路线规划

系统根据项目类型和行业骨架推荐生成路线。用户选择合适路线后确认。

如果路线不合适，可回到系统设置调整行业骨架或重新生成。

### 5.4 目录生成

系统生成技术标目录。用户重点检查：

- 是否覆盖评分表必出项
- 是否覆盖强制响应条款
- 是否避免无关章节
- 目录层级是否符合投标文件结构

### 5.5 目录校验

系统会检查目录覆盖度、冲突项和风险项。用户需要处理：

- 必出项缺失
- 章节重复
- 章节名称与招标要求不一致
- 关键要求只在父章节出现、没有独立响应

校验通过后进入正文生成。

### 5.6 正文生成

系统按目录生成技术标正文。用户检查：

- 每章是否有实质内容
- 是否贴合项目画像
- 是否引用正确事实资料
- 是否有空泛、重复或模板化内容

### 5.7 风控合规

系统对整标内容做风险复核。用户重点查看：

- 废标风险
- 与招标文件冲突的描述
- 资质、人员、业绩事实不一致
- 工期、质量、安全承诺不一致

### 5.8 导出定稿

风险处理后进入“导出定稿”，生成 Word/PDF 或成果包。导出后仍建议人工做最终排版和盖章检查。

## 6. 审计与异常中心

异常中心用于查看：

- 文件解析失败
- 资料缺失
- 风险审计记录
- 流程中断
- AI 生成失败

处理建议：

1. 先查看错误信息。
2. 回到对应业务页面修正资料或配置。
3. 重新执行当前步骤。

## 7. 常见提示说明

### “缺少 API Key”

系统设置里未配置大模型 Key。填写后保存，再重试当前步骤。

### “OCR 引擎未就绪”

OCR 服务未启动或地址错误。检查系统设置中的 OCR 服务地址。

### “请先生成并确认数据块”

商务标“方案处理确认”还没完成。回退到该步骤，逐章确认装配资料。

### “无可下载文件”

导出文件不存在或已被清理。重新执行导出。

### “处理失败”

查看异常中心或后端日志。常见原因是 API Key 无效、文档解析失败、上传文件无法读取、前置步骤未确认。

## 8. 数据安全提醒

招标文件、投标文件、证照、人员信息、业绩合同和导出成果通常包含敏感信息。请遵守以下原则：

- 不要把运行数据库发到公开仓库。
- 不要把上传文件、导出 Word、PDF 发到公开仓库。
- 不要在截图里暴露 API Key 或人员身份证号。
- 对外发送前先脱敏。
