# 开发部署手册

本文面向接手开发、部署演示和二次开发的人，说明如何从源码启动项目、配置运行环境、执行检查和排查常见问题。

## 1. 环境要求

- Go 1.22 或更高版本
- Node.js 20 或更高版本
- npm 10 或更高版本
- macOS、Linux 或 Windows WSL 均可
- 可选：本地 OCR 服务、阿里云文档智能、兼容 OpenAI/DashScope 的大模型接口

## 2. 克隆与目录

```bash
git clone https://github.com/Bronc-X/public-tender.git
cd public-tender
```

核心目录：

```text
backend_go/      Go 后端
frontend/        React 前端
docs/            设计文档、操作手册、验证日志
data/            本地运行数据，默认不会进入 Git
```

## 3. 后端启动

```bash
cd backend_go
go mod download
PORT=8081 APP_DB_PATH=data/app.db go run ./cmd/server
```

环境变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `PORT` | `8081` | Go API 服务端口 |
| `APP_DB_PATH` | `data/app.db` | SQLite 数据库路径 |
| `NODE_BACKEND` | `http://localhost:8889` | 未命中 Go 路由时的兼容代理目标 |
| `BID_EXPORT_DIR` | `backend_go/data/exports/bid_projects` | 商务标导出目录 |
| `TECH_BID_EXPORT_DIR` | `backend_go/data/exports/tech_bid_projects` | 技术标导出目录 |

健康检查：

```bash
curl http://127.0.0.1:8081/health
```

数据库说明：

- 首次启动会自动创建 SQLite 数据库。
- schema 创建和补丁逻辑在 `backend_go/internal/db/` 与 `backend_go/migrations/`。
- 运行数据库、上传文件、导出文件都在 `data/` 或 `backend_go/data/` 下，默认不提交。

## 4. 前端启动

```bash
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

访问：

```text
http://127.0.0.1:5173/
```

前端开发代理在 `frontend/vite.config.ts`：

- `/api` -> `http://localhost:8081`
- `/files` -> `http://localhost:8081`

## 5. 系统配置

进入前端后打开“系统设置”。

常用配置项：

| 配置 | 用途 |
| --- | --- |
| `ai_api_key` | 商务标和技术标生成使用的大模型 Key |
| `ai_ingest_endpoint` | 兼容 OpenAI/DashScope 的模型服务地址 |
| `ai_ingest_model` | 模型名称，默认可使用 Qwen/DashScope 兼容模型 |
| `doc_parser_access_key` | 阿里云文档智能 Access Key |
| `doc_parser_access_secret` | 阿里云文档智能 Secret |
| OCR 设置 | 本地 OCR 或高级 OCR 服务地址 |

不要把任何 Key 写进源码、README、提交记录或截图。

## 6. 构建与测试

后端：

```bash
cd backend_go
go test ./...
```

前端：

```bash
cd frontend
npm run build
```

`npm run build` 会先执行 TypeScript 构建，再执行 Vite 生产构建。

## 9. 发布到 GitHub 前检查

```bash
git status --short
git ls-files | grep -E 'node_modules|dist|data/|\\.db$|\\.log$|\\.zip$' || true
cd backend_go && go test ./...
cd ../frontend && npm run build
```

应确认以下内容没有被提交：

- 数据库：`*.db`、`*.sqlite`
- 运行数据：`data/`、`backend_go/data/`
- 上传文件和导出文件
- 日志：`*.log`
- 构建产物：`dist/`
- 依赖目录：`node_modules/`
- 编译二进制：`server`、`main`、`server_bin`
- 本地设置：`.claude/settings.local.json`

## 10. 常见问题

### 前端请求 404 或接口不通

确认后端在 `8081` 端口运行：

```bash
curl http://127.0.0.1:8081/health
```

确认前端通过 Vite dev server 访问，不要直接打开 `index.html`。

### 提示缺少 API Key

进入“系统设置”，填写并保存大模型配置。保存后重新执行当前步骤。

### 文档解析失败

检查以下配置：

- 阿里云文档智能 Key/Secret 是否配置
- OCR 服务是否启动
- 上传文件格式是否为 PDF/DOCX/图片
- 后端日志是否有网络或鉴权错误

### 商务标 Step6 提示“请先生成/确认数据块”

回到“方案处理确认”，逐章完成“章节装配参数”配置，再进入“标书章节生成”。如果项目已经进入 Step6 错误态，点击页面按钮“回到方案处理确认”或右上角“回退上一步”。

### GitHub 推送超时

本机网络直连 GitHub 可能超时。若使用 Clash Verge，可用：

```bash
git -c http.proxy=http://127.0.0.1:7897 \
    -c https.proxy=http://127.0.0.1:7897 \
    -c http.version=HTTP/1.1 \
    push
```

端口以本机代理配置为准。
