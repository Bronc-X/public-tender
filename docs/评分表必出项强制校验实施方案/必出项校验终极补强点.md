# 必出项校验终极补强点

> 位置：与《评分表必出项强制校验实施方案》同目录
> 目的：在现有“必出项提取 + 硬校验 + 自动补章”基础上，再补 3 个关键硬点

---

## 一、补强点 1：必出项不要只从 facts 里推，要尽量前移到评分表原文

### 当前问题
现在 `mandatoryChapters` 主要还是从 `facts` 里推出来的。  
这意味着：
- 如果 facts 抽漏了“环境保护管理体系与措施”
- 后面的强校验也会一起漏掉

### 应该怎么改
在 facts 之前，或至少和 facts 并行，增加一层：

**评分表独立项抽取**

直接从以下章节原文中提取：
- 综合评分明细表
- 评分标准
- 评标办法
- 技术评分标准

### 输出结果
形成一个单独的数据结构，例如：

```go
type MandatoryChapterSpec struct {
    Name string   // 如：环境保护管理体系与措施
    Source string // 来自评分表第几项
    MustBeTopLevel bool // 是否必须一级章
}
```

### 为什么这一步重要
因为真正决定“必须独立成章”的，不是 facts 摘要，而是**评分表本身**。

### 结论
**必出项来源前移，是下一步最重要的补强。**

---

## 二、补强点 2：从“关键词匹配”升级成“评分项 → 一级章”的结构化映射

### 当前问题
现在 `ValidateMandatoryChapters` 主要还是：
- 看章名是否包含关键词
- 看章名是否模糊命中

这虽然能兜底，但不够硬：
- 改写章名后可能误判
- 合并章节后可能误判通过
- 相似表达可能被错判

### 应该怎么改
建立一个更明确的结构化映射：

```go
type MandatoryChapterRule struct {
    CanonicalName string
    Aliases []string
    MustBeTopLevel bool
}
```

例如：

```go
{
  CanonicalName: "环境保护管理体系与措施",
  Aliases: []string{"环境保护", "环保管理", "环境管理"},
  MustBeTopLevel: true,
}
```

### 校验规则升级为
1. 评分项必须命中 `CanonicalName` 或 `Aliases`
2. 命中位置必须是 **一级章**
3. 如果只在二级节/三级小节出现，不算通过
4. 如果被并入“安全文明施工”一章，也不算通过

### 结论
**以后校验不要再只看“像不像”，要看“是不是明确对应到独立一级章”。**

---

## 三、补强点 3：补章逻辑不要只在末尾加占位，要有“定位插入 + 循环校验”

### 当前问题
现在 `PatchMissingMandatoryChapters` 的方式是：
- 缺什么，就在末尾追加一个一级章
- 里面塞一个固定占位节和占位小节

这能防漏，但有 3 个问题：
1. 追加位置不一定合理
2. 和原目录逻辑可能不协调
3. 补完后只再校验一次，不够稳

### 应该怎么改

#### 1）按类别决定插入位置
例如：
- 质量管理体系与措施 → 放在施工方案后
- 安全文明施工管理体系与措施 → 放在质量管理后
- 环境保护管理体系与措施 → 放在安全文明之后
- 工期保证措施 → 放在资源配备前后，按既定规范插入

也就是：
**不是一律追加到末尾，而是按常见技术标骨架插入到合理位置。**

#### 2）补章后循环校验
建议改成：
- 最多循环 2~3 轮
- 每轮：校验 → 补章 → 再校验
- 直到没有缺失，或达到最大轮数

伪代码：

```go
for i := 0; i < 3; i++ {
    missing, ok := ValidateMandatoryChapters(outline, mandatoryChapters)
    if ok {
        break
    }
    outline = PatchMissingMandatoryChapters(outline, missing)
}
```

#### 3）补章后再次做骨架漂移检查
补章完成后，和原始目录做一次骨架对比：
- 确保没有把原有一级章打乱
- 确保只是插入缺失章，而不是整篇改坏

### 结论
**补章不应该只是“加一个壳子”，而应该是“按结构插进去，并校验到稳定”。**

---

## 四、最终建议顺序

### 优先级 P0（必须做）
1. 必出项来源前移到评分表原文
2. 评分项 → 一级章 映射结构化

### 优先级 P1（应该做）
3. 补章按合理位置插入
4. 补章改成循环校验到稳定

### 优先级 P2（增强项）
5. 补章后再次做骨架漂移检查
6. 为每个必出项记录来源页码 / 来源位置，方便前端展示给用户

---

## 五、一句话总结

当前这版已经能防大部分漏章，但如果要做到更稳，必须再补这 3 点：

1. **必出项前移到评分表原文抽取**
2. **评分项到一级章的结构化映射**
3. **补章按位置插入 + 循环校验到稳定**
