# 代码层面：必出项硬校验 — 程序员完整实施文档

> CTO 出品 | 2026-04-10
> 目标：在目录生成后，通过代码强制校验评分表独立项是否都独立成章，缺了就自动补，不依赖提示词

---

## 一、背景

提示词已经写了"独立项必须独立成章"。但模型不一定 100% 遵守。
所以代码必须做兜底：**生成完目录后，立刻检查，缺了就补。**

---

## 二、需要新增的 4 个函数

全部放在 `backend_go/internal/service/tender_digitization.go`

---

### 函数 1：ExtractMandatoryChapters

#### 作用
从 facts 中筛选出"必须独立成章"的条目。

#### 输入
`facts *FactExtractResult`

#### 输出
`[]string`，例如：
```go
["施工方案和技术措施", "质量管理体系与措施", "环境保护管理体系与措施", ...]
```

#### 筛选规则
遍历 `facts.ScoreItems` 和 `facts.MandatorySpecs`，凡是名称包含以下关键词的，收录：

```
施工方案、技术措施
质量管理
安全文明、安全管理
环境保护
工期保证、工期计划、工期
资源配备、资源配置
施工总平面、平面布置
项目管理机构
```

#### 注意
- 去重
- 排序（保证顺序稳定）
- 如果 facts 为 nil，返回空

---

### 函数 2：ValidateMandatoryChapters

#### 作用
检查生成的目录一级章里，是否包含了所有必出项。

#### 输入
- `outline []map[string]interface{}`（生成的目录）
- `mandatoryChapters []string`（必出项清单）

#### 输出
- `missing []string`（缺失的必出项）
- `ok bool`（是否全部通过）

#### 校验规则
遍历每个必出项，检查是否在目录的某个一级章名称中命中。

命中方式（宽松匹配）：
1. 章名直接包含必出项名称
2. 章名包含必出项的别名关键词（见下方映射表）

#### 结构化映射表
用于别名匹配，避免因标题改写导致误判：

| 必出项关键词 | 别名列表 |
|-------------|---------|
| 施工方案 | 施工方案、技术措施、技术方案 |
| 质量管理 | 质量管理、质量保证、质量保障 |
| 安全文明 | 安全文明、安全施工、安全管理 |
| 环境保护 | 环境保护、环保管理、环境管理、绿色施工 |
| 工期保证 | 工期、进度、工期保证 |
| 资源配备 | 资源、人机料、资源配置 |
| 平面布置 | 平面布置、施工总平面、总平面 |
| 项目管理机构 | 项目管理机构、项目组织、管理架构 |

#### 重要
- **只看一级章**，不看二级节、三级小节
- 如果"环境保护"只出现在某个章的二级节里，不算通过

---

### 函数 3：PatchMissingMandatoryChapters

#### 作用
对缺失的必出项，自动补一个一级章。

#### 输入
- `outline []map[string]interface{}`（当前目录）
- `missing []string`（缺失的必出项名称列表）

#### 输出
`[]map[string]interface{}`（补章后的目录）

#### 补章规则

##### 章结构
每个缺失项补一个一级章，结构如下：
```json
{
  "name": "第X章 环境保护管理体系与措施",
  "units": [
    {
      "name": "第一节 总体措施",
      "subsections": [
        {"name": "一、待补充内容", "requirement_ids": []}
      ]
    }
  ]
}
```

##### 插入位置
不要一律追加到末尾。按以下顺序插入：

| 必出项 | 应插入在...之后 |
|--------|---------------|
| 施工方案和技术措施 | 目录开头 |
| 质量管理体系与措施 | 施工方案和技术措施 |
| 安全文明施工管理体系与措施 | 质量管理体系与措施 |
| 环境保护管理体系与措施 | 安全文明施工管理体系与措施 |
| 工期保证措施 | 环境保护管理体系与措施 |
| 资源配备计划 | 工期保证措施 |
| 施工总平面布置 | 资源配备计划 |
| 项目管理机构配置 | 施工总平面布置 |

如果找不到"应插入在...之后"的那个章，就追加到末尾。

---

### 函数 4：PatchMissingMandatoryChaptersWithLoop

#### 作用
循环执行"校验 → 补章"，直到没有缺失项，或达到最大轮数。

#### 输入
- `outline []map[string]interface{}`
- `mandatoryChapters []string`
- `maxLoops int`（建议传 3）

#### 输出
- `[]map[string]interface{}`（最终目录）
- `bool`（是否全部通过）

#### 逻辑
```go
for i := 0; i < maxLoops; i++ {
    missing, ok := ValidateMandatoryChapters(outline, mandatoryChapters)
    if ok {
        return outline, true
    }
    log.Printf("循环校验第 %d 轮：缺失 %v，继续补章", i+1, missing)
    outline = PatchMissingMandatoryChapters(outline, missing)
}
// 最后再校验一次
missing, ok := ValidateMandatoryChapters(outline, mandatoryChapters)
if !ok {
    log.Printf("⚠️ 循环补章后仍有缺失: %v", missing)
}
return outline, ok
```

    if !ok {
        log.Printf("[Step4] 必出项缺失: %v，启动循环补章", missing)

    mandatoryText = "【强制独立成章项（不得合并到其他章节，必须独立列为一级章）】\n"

dynamicInstruction := fmt.Sprintf(`### 以下为评分表中必须独立成章的条目（硬性约束，不得合并）
%s

### 以下为从招标文件中预抽取的评分项和强制项摘要（仅供辅助参考，招标文件全文才是唯一真相源）

## 七、验收标准

1. 目录生成后，如果缺少"环境保护管理体系与措施"等必出项，系统自动补章
2. 补章位置合理（不是一律追加到末尾）
3. 补章后会再次校验，最多循环 3 轮
4. 日志里能看到：
   - `[Step4] 必出项缺失: [xxx]，启动循环补章`
   - `[Step4] 循环补章后目录共 X 个一级章`
   - 或 `[Step4] 必出项校验通过，无需补章`
5. `go build ./...` 编译通过

## 九、风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| 必出项关键词匹配不准 | 低 | 用别名映射表做宽松匹配 |
| 自动补章的占位内容太空 | 低 | 后续正文撰写阶段填充 |
| 不同招标文件评分项名称差异大 | 中 | 必出项从 facts 动态提取，不完全写死 |
| 循环补章死循环 | 极低 | maxLoops 限制为 3 |
