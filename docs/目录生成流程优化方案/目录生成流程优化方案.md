# 目录生成流程优化方案

> CTO 出品 | 2026-04-10

---

## 一、现状（8步）

```
第1步：招标解析（上传招标文件）
第2步：项目画像（独立调大模型提取）
第3步：Facts 提取（评分项、强制项等）
第4步：路线规划（用户选择，但没真正参与生成）
第5步：目录生成（喂了太多东西，效果差）
第6步：审计校验
第7步：补全优化（可能把目录改差）
第8步：终审确认
```

### 主要问题
1. 项目画像独立调大模型，但和 facts 内容高度重复
2. 路线规划选了但没真正参与目录生成
3. 目录生成输入太杂，模型容易跑偏
4. 后链路可能把好目录改差

---

## 二、优化后（6步）

```
第1步：招标解析（不变）
第2步：Facts 提取 + 可视化展示（合并原来的 2、3 步）
第3步：路线规划（用户选择，真正参与生成）
第4步：目录生成（精准投喂）
第5步：审计校验（只发现问题，不重写）
第6步：补全 + 终审确认（只补洞，不重构）
```

---

## 三、每一步具体说明

### 第1步：招标解析（不变）
- 用户上传招标文件
- 系统解析成文本
- 无改动

### 第2步：Facts 提取 + 可视化展示
- 从招标文件提取：评分项、强制项、特殊技术点
- 提取结果按重要性排序
- 前端直接展示 facts 内容（替代原来的项目画像）
- 着重展示：评分项、强制项、特殊技术点
- 不再单独调大模型生成项目画像

### 第3步：路线规划
- 用户选择写作风格
- 比如：常规施工组织 / 专项攻坚 / 竞争性表达
- 这个选择会真正传给目录生成环节，影响目录风格

### 第4步：目录生成（核心改动）
喂给大模型的输入，只有 4 样：

| 输入 | 作用 |
|------|------|
| 招标文件原文 | 主材料，内容来源 |
| 完整提示词 | 生成规则 |
| 路线规划 | 写作风格引导 |
| 少量高权重 facts | 重点提醒（只取最重要的评分项、强制项） |

不喂：项目画像、全量 facts、全量 requirements、industry hint

### 第5步：审计校验
- 只输出：缺了什么、哪里弱、哪里有风险
- 不动目录本身
- 定位是"质检员"，不是"第二个设计师"

### 第6步：补全 + 终审确认
- 只允许在指定位置插入缺失小节
- 一级章、二级节冻结不动
- 三级小节允许增补和微调
- 偏移过大自动回滚
- 最终展示给用户确认

---

## 四、删掉/合并了什么

| 原来 | 现在 |
|------|------|
| 项目画像（独立步骤） | 合并到 facts 展示，不再单独调大模型 |
| 路线规划（摆设） | 真正接入目录生成 |
| 全量 facts / requirements 喂给生成 | 只喂少量高权重摘要 |
| 后链路可能重写目录 | 加了骨架冻结 + 回滚保护 |

---

## 五、核心收益

1. **少调一次大模型**（省钱、省时间）
2. **目录生成效果更好**（输入更纯净）
3. **流程更短**（8步 → 6步）
4. **用户体验更清晰**（看到的就是模型用到的）
5. **后链路不会把好目录改差**（已加保护）

---

## 六、一句话总结

**把流程从"多步重复、一锅炖"改成"精准投喂、只补不改"。**
